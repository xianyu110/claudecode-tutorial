# 2.6 连接本地开发环境

## 2.6.1 概述#

Claude Code需要连接到本地开发环境才能访问和修改代码文件。连接本地开发环境是使用Claude Code的重要步骤，它确保Claude Code能够在你的开发环境中正常工作。本节将介绍如何连接Claude Code到本地开发环境，包括权限设置、工作目录配置等。

## 2.6.2 连接步骤#

### 2.6.2.1 在VS Code中连接#

  1. 打开VS Code
  2. 打开你要工作的项目文件夹（**文件** **打开文件夹** ）
  3. 点击左侧边栏的**Claude Code** 图标
  4. 如果是首次使用，Claude Code会提示你授予访问权限
  5. 点击**授权** 按钮，允许Claude Code访问你的项目文件夹
  6. 授权成功后，Claude Code会显示项目的文件结构

### 2.6.2.2 在命令行工具中连接#

  1. 打开终端或命令提示符
  2. 导航到你的项目目录

         bash

         复制

         cd /path/to/your/project

  3. 运行Claude Code命令

         bash

         复制

         claude chat

  4. 如果是首次使用，Claude Code会提示你授予访问权限
  5. 输入`y`或`yes`，允许Claude Code访问当前目录
  6. 授权成功后，Claude Code会显示项目的文件结构

## 2.6.3 权限设置#

Claude Code需要以下权限才能正常工作：

### 2.6.3.1 文件系统权限#

  * **读取权限** ：读取项目文件和目录结构
  * **写入权限** ：修改和创建项目文件
  * **执行权限** ：运行测试和构建命令

### 2.6.3.2 权限管理#

  1. **最小权限原则** ：Claude Code遵循最小权限原则，只请求必要的权限
  2. **用户确认** ：所有可能修改文件的操作都需要用户确认
  3. **权限撤销** ：可以随时撤销Claude Code的权限

### 2.6.3.3 安全边界#

Claude Code在安全边界内运行，确保：

  * 不会访问项目文件夹外的文件
  * 不会执行危险的系统命令
  * 所有操作都有详细的日志记录

## 2.6.4 工作目录配置#

### 2.6.4.1 设置默认工作目录#

可以配置Claude Code的默认工作目录，方便快速访问常用项目：

  1. **在VS Code中** ：

     * 打开VS Code设置
     * 搜索 "Claude Code"
     * 找到"Default Workspace"设置
     * 输入你常用的项目目录路径
  2. **在命令行工具中** ：

         bash

         复制

         claude config set default-workspace /path/to/your/project

### 2.6.4.2 切换工作目录#

在命令行工具中，可以使用以下命令切换工作目录：

    bash

    复制

    # 切换到指定目录
    claude cd /path/to/your/project
    # 列出当前目录内容
    claude ls
