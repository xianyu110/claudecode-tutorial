# 7.8 /config - 開啟設定介面

## 概述
`/config` 命令用於開啟 Claude Code 的設定介面，允許你配置各種選項和偏好設定。設定介面提供了圖形化的配置方式，方便使用者自定義 Claude Code 的行為。

```
## 基本语法

    bash


    /config

## 功能说明

```
執行 `/config` 命令後，Claude Code 會：

  1. 開啟設定介面（配置選項卡）
  2. 顯示所有可配置的選項
  3. 允許使用者修改配置
  4. 儲存配置更改

## 配置選項
設定介面包含以下主要配置類別：

### 1\. 模型設定
  * 選擇預設 AI 模型
  * 配置模型引數
  * 設定溫度、最大令牌數等

### 2\. 介面設定
  * 主題選擇
  * 字型大小
  * 介面佈局

### 3\. 行為設定
  * 自動儲存
  * 檔案監聽
  * 上下文管理

### 4\. 整合設定
  * Git 整合
  * MCP 伺服器
  * 外掛配置

### 5\. 安全設定
  * 許可權管理
  * 隱私設定
  * 沙箱配置

```
## 使用场景

### 1\. 更改默认模型

    bash


    /config

    # 在设置界面中：
    # 1. 选择"模型"选项卡
    # 2. 选择默认模型
    # 3. 保存更改

### 2\. 调整界面主题

    bash


    /config

    # 在设置界面中：
    # 1. 选择"界面"选项卡
    # 2. 选择主题
    # 3. 保存更改

### 3\. 配置权限

    bash


    /config

    # 在设置界面中：
    # 1. 选择"安全"选项卡
    # 2. 配置权限规则
    # 3. 保存更改

## 注意事项

```
  1. 配置更改會立即生效
  2. 某些配置可能需要重啟 Claude Code 才能完全生效
  3. 配置檔案儲存在使用者目錄中
  4. 可以手動編輯配置檔案進行高階配置

## 配置檔案位置
配置檔案通常位於：

  * **Windows** : `%APPDATA%\claude\config.json`
  * **macOS** : `~/Library/Application Support/claude/config.json`
  * **Linux** : `~/.config/claude/config.json`

```
## 与其他命令的配合使用

    bash


    # 打开配置，然后验证更改
    /config
    /doctor

    # 执行流程：
    # 1. 打开设置界面
    # 2. 修改配置
    # 3. 运行诊断检查验证配置

## 最佳实践

```
  1. **定期檢查** ：定期檢查配置設定，確保符合當前需求
  2. **備份配置** ：在大幅修改配置前，備份配置檔案
  3. **測試更改** ：修改配置後，測試相關功能是否正常
  4. **團隊共享** ：將團隊常用的配置共享到專案中

```

```

  * 開啟圖形化設定介面
  * 配置各種選項和偏好
