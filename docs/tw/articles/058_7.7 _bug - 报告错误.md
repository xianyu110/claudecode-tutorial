# 7.7 /bug - 報告錯誤

## 概述

`/bug` 命令用於向 Anthropic 報告 Claude Code 的錯誤或問題。執行此命令會將當前對話內容傳送給 Anthropic 團隊，幫助他們診斷和修復問題。

## 基本语法

    bash


    /bug

## 功能说明


執行 `/bug` 命令後，Claude Code 會：

  1. 收集當前會話的對話歷史
  2. 收集系統資訊和錯誤日誌
  3. 將這些資訊傳送給 Anthropic 團隊
  4. 顯示報告提交確認資訊

```
## 使用场景

### 1\. 遇到程序错误

    bash


    # 当 Claude Code 出现异常或错误时
    /bug

    # 系统会收集错误信息并发送给 Anthropic

### 2\. 功能异常

    bash


    # 当某个功能不按预期工作时
    /bug

    # 报告功能问题，帮助改进产品

### 3\. 性能问题

    bash


    # 当遇到性能问题时
    /bug

    # 报告性能瓶颈，帮助优化

## 报告内容

```

`/bug` 命令會收集以下資訊：

  1. **對話歷史** ：當前會話的完整對話內容
  2. **系統資訊** ：作業系統、Claude Code 版本等
  3. **錯誤日誌** ：相關的錯誤資訊和堆疊跟蹤
  4. **環境資訊** ：工作目錄、檔案結構等

## 注意事項

  1. 報告會包含當前對話的完整內容，請確保不包含敏感資訊
  2. 報告是匿名的，不會關聯到你的賬戶
  3. 提交報告後，Anthropic 團隊會進行分析和修復
  4. 不要濫用此命令，只在遇到真正的問題時使用
  5. 報告後無法撤銷，請謹慎使用

## 隱私保護


  * 對話內容僅用於問題診斷
  * 不會用於訓練 AI 模型
  * 不會與第三方共享
  * 資料會按照 Anthropic 的隱私政策處理

## 与其他命令的配合使用

    bash


    # 遇到问题时，先尝试诊断，然后报告
    /doctor
    /bug

    # 执行流程：
    # 1. 运行诊断检查
    # 2. 如果问题无法解决，提交错误报告

## 最佳实践


  1. **先診斷** ：在使用 `/bug` 前，先嚐試 `/doctor` 診斷問題
  2. **描述清楚** ：在報告前，確保對話中清楚描述了問題
  3. **避免敏感資訊** ：確保對話中不包含密碼、金鑰等敏感資訊
  4. **合理使用** ：只在遇到真正的 bug 時使用，不要濫用

## 何時使用 /bug

**適合使用的情況** ：

  * Claude Code 崩潰或無響應
  * 功能完全不工作
  * 出現明顯的程式錯誤
  * 效能嚴重下降

**不適合使用的情況** ：

  * AI 回答不準確（這是模型能力問題）
  * 功能不符合預期（這是使用問題）
  * 需要功能建議（這是功能請求）

## 總結

`/bug` 命令是 Claude Code 中用於報告錯誤的重要工具。透過使用此命令，你可以：

  * 向 Anthropic 團隊報告程式錯誤
  * 幫助改進 Claude Code 的質量
  * 促進產品的持續最佳化

合理使用 `/bug` 命令可以幫助 Anthropic 團隊更快地發現和修復問題，提升所有使用者的體驗。
