# 2.6 連線本地開發環境

## 2.6.1 概述

Claude Code需要連線到本地開發環境才能訪問和修改程式碼檔案。連線本地開發環境是使用Claude Code的重要步驟，它確保Claude Code能夠在你的開發環境中正常工作。本節將介紹如何連線Claude Code到本地開發環境，包括許可權設定、工作目錄配置等。

## 2.6.2 连接步骤

### 2.6.2.1 在VS Code中连接


  1. 開啟VS Code
  2. 開啟你要工作的專案資料夾（**檔案** **開啟資料夾** ）
  3. 點選左側邊欄的**Claude Code** 圖示
  4. 如果是首次使用，Claude Code會提示你授予訪問許可權
  5. 點選**授權** 按鈕，允許Claude Code訪問你的專案資料夾
  6. 授權成功後，Claude Code會顯示專案的檔案結構

### 2.6.2.2 在命令列工具中連線

  1. 開啟終端或命令提示符
  2. 導航到你的專案目錄

```bash
         bash


         cd /path/to/your/project

```

  3. 執行Claude Code命令

```
         bash


         claude chat

```

  4. 如果是首次使用，Claude Code會提示你授予訪問許可權
  5. 輸入`y`或`yes`，允許Claude Code訪問當前目錄
  6. 授權成功後，Claude Code會顯示專案的檔案結構

## 2.6.3 許可權設定

Claude Code需要以下許可權才能正常工作：

### 2.6.3.1 檔案系統許可權


  * **讀取許可權** ：讀取專案檔案和目錄結構
  * **寫入許可權** ：修改和建立專案檔案
  * **執行許可權** ：執行測試和構建命令

### 2.6.3.2 許可權管理

  1. **最小許可權原則** ：Claude Code遵循最小許可權原則，只請求必要的許可權
  2. **使用者確認** ：所有可能修改檔案的操作都需要使用者確認
  3. **許可權撤銷** ：可以隨時撤銷Claude Code的許可權

### 2.6.3.3 安全邊界

Claude Code在安全邊界內執行，確保：

  * 不會訪問專案資料夾外的檔案
  * 不會執行危險的系統命令
  * 所有操作都有詳細的日誌記錄

## 2.6.4 工作目录配置

### 2.6.4.1 设置默认工作目录


可以配置Claude Code的預設工作目錄，方便快速訪問常用專案：

  1. **在VS Code中** ：

```
     * 打开VS Code设置
     * 搜索 "Claude Code"
     * 找到"Default Workspace"设置
     * 输入你常用的项目目录路径

```

  2. **在命令列工具中** ：

```
         bash


         claude config set default-workspace /path/to/your/project

### 2.6.4.2 切换工作目录

```

在命令列工具中，可以使用以下命令切換工作目錄：

```bash
    bash


    # 切换到指定目录
    claude cd /path/to/your/project
    # 列出当前目录内容
    claude ls

```
