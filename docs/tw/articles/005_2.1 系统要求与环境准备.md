# 2.1 系統要求與環境準備

## 2.1.1 系統要求

在安裝Claude Code之前，確保你的系統滿足以下要求：

### 2.1.1.1 作業系統


  * **Windows** : Windows 10 或更高版本，64位
  * **macOS** : macOS 10.15 (Catalina) 或更高版本
  * **Linux** : Ubuntu 18.04 或更高版本，或其他相容的Linux發行版

### 2.1.1.2 硬體要求


  * **CPU** : 至少4核處理器
  * **記憶體** : 至少8GB RAM（推薦16GB或更高）
  * **磁碟空間** : 至少1GB可用空間
  * **網路** : 穩定的網際網路連線（用於授權驗證和功能更新）

### 2.1.1.3 軟體要求


  * **Node.js** : 版本16.0或更高（用於命令列工具）
  * **npm** : 版本8.0或更高（Node.js包管理器）
  * **Git** : 版本2.0或更高（用於程式碼管理和外掛安裝）

## 2.1.2 环境准备

### 2.1.2.1 安装Node.js和npm


Claude Code的命令列工具依賴Node.js和npm，因此需要先確保它們已正確安裝。

#### 檢查Node.js和npm版本

開啟終端或命令提示符，執行以下命令檢查Node.js和npm版本：

```
    bash


    # 检查Node.js版本
    node --version

    # 检查npm版本
    npm --version


如果輸出顯示Node.js版本≥16.0和npm版本≥8.0，則可以跳過安裝步驟。

#### 安裝Node.js

如果沒有安裝Node.js或版本不符合要求，可以透過以下方式安裝：

##### 方式一：使用官方安裝包

  1. 訪問Node.js官方網站：<https://nodejs.org/>
  2. 下載適合你作業系統的LTS（長期支援）版本
  3. 執行安裝程式，按照提示完成安裝
  4. 安裝完成後，重新開啟終端，驗證Node.js和npm版本

##### 方式二：使用Node版本管理器（推薦）

使用Node版本管理器可以方便地管理多個Node.js版本，推薦使用nvm（Node Version Manager）： **對於macOS和Linux：**

    bash


    # 安装nvm
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.5/install.sh | bash
    # 或使用wget
    wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.5/install.sh | bash

    # 安装完成后，重新打开终端，安装Node.js LTS版本
    nvm install --lts

    # 验证安装
    node --version
    npm --version

    nvm install lts
    nvm use lts
    # 验证安装
    node --version
    npm --version

### 2.1.2.2 安装Git


Claude Code使用Git進行程式碼管理和外掛安裝，因此需要確保Git已正確安裝。

#### 檢查Git版本

開啟終端或命令提示符，執行以下命令檢查Git版本：

    bash


    git --version


如果輸出顯示Git版本≥2.0，則可以跳過安裝步驟。

#### 安裝Git

**對於macOS：**

macOS系統可能已預裝Git，你可以透過以下方式更新到最新版本：

    bash


    # 使用Homebrew安装（推荐）
    brew install git
    # 或使用Xcode Command Line Tools
    xcode-select --install


**對於Windows：**

  1. 訪問Git官方網站：<https://git-scm.com/>
  2. 下載適合Windows的Git安裝程式
  3. 執行安裝程式，按照提示完成安裝（建議使用預設配置） **對於Linux：** 使用包管理器安裝Git：

    bash


    # Ubuntu/Debian
    sudo apt-get update
    sudo apt-get install git

    # CentOS/RHEL
    sudo yum install git

    # Fedora
    sudo dnf install git

### 2.1.2.3 配置Git

```

安裝Git後，需要配置使用者名稱和電子郵件，這些資訊將用於程式碼提交：

```
    bash


    git config --global user.name "Your Name"
    git config --global user.email "your.email@example.com"

## 2.1.3 环境验证

```

完成上述安裝後，執行以下命令驗證環境是否準備就緒：

```
    bash


    # 验证Node.js版本
    node --version

    # 验证npm版本
    npm --version

    # 验证Git版本
    git --version

```

如果所有命令都能正常輸出且版本符合要求，則說明環境準備就緒，可以開始安裝Claude Code了。

## 2.1.4 常见问题与解决方案

### 2.1.4.1 Node.js安装失败


**問題** ：執行`node --version`時顯示"command not found"

**解決方案** ：

  * 檢查安裝過程是否有錯誤
  * 確保Node.js的安裝路徑已新增到系統環境變數中
  * 重新啟動終端或命令提示符
  * 嘗試使用Node版本管理器重新安裝

### 2.1.4.2 npm安裝包失敗

**問題** ：使用npm安裝包時顯示網路錯誤或許可權錯誤

**解決方案** ：

  * 檢查網路連線
  * 嘗試使用npm映象（如淘寶映象）：`npm config set registry https://registry.npmmirror.com`
  * 對於許可權錯誤，嘗試使用`sudo`（Linux/macOS）或以管理員身份執行命令提示符（Windows）
  * 檢查npm配置：`npm config list`

### 2.1.4.3 Git配置錯誤

**問題** ：Git提交時顯示使用者名稱或電子郵件未配置

**解決方案** ：

  * 確保已正確配置Git使用者名稱和電子郵件
  * 檢查配置：`git config --list`
  * 如果全域性配置不起作用，嘗試在專案目錄中進行本地配置

## 2.1.5 高级环境配置

### 2.1.5.1 使用yarn或pnpm替代npm


如果你更喜歡使用yarn或pnpm作為包管理器，Claude Code也支援它們：

**安裝yarn：**

```
    bash


    npm install -g yarn


**安裝pnpm：**

    bash


    npm install -g pnpm

### 2.1.5.2 配置npm镜像

```

為了提高npm包的下載速度，可以配置npm映象：

```
    bash


    # 使用淘宝镜像
    npm config set registry https://registry.npmmirror.com
    # 或使用npm官方镜像
    npm config set registry https://registry.npmjs.org/

### 2.1.5.3 配置HTTP代理

```

如果你的網路環境需要使用HTTP代理，可以配置npm和Git的代理設定： **配置npm代理：**

```
    bash


    # 设置HTTP代理
    npm config set proxy http://proxy.example.com:8080
    npm config set https-proxy http://proxy.example.com:8080

    # 取消代理设置
    npm config delete proxy
    npm config delete https-proxy


**配置Git代理：**

    bash


    # 设置HTTP代理
    git config --global http.proxy http://proxy.example.com:8080
    git config --global https.proxy http://proxy.example.com:8080
    # 取消代理设置
    git config --global --unset http.proxy
    git config --global --unset https.proxy

```
